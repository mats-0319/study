# utils

- generate random bytes：生成可读的随机字符串
  - 字符库索引法：定义一个随机字符库，生成随机的索引，使用索引访问字符库
  - 字节编码法：生成一串随机的数，选择一种编码方式保证其可读
- generate random slice：生成指定长度的泛型slice，slice元素的值随机
- uuid v4：google uuid，新增**输入相同参数，输出相同id**功能
- uuid v7：一定程度上有序，可用作数据库记录id     				
       				
## 数据库记录id类型
       	 
| 维度       | 自增序列 (Auto-increment)     | UUID (Universally Unique Identifier) |
|----------|---------------------------|--------------------------------------|
| 存储大小     | 小（4 字节 INT 或 8 字节 BIGINT） | 大（通常 16 字节或 36 字节字符串）                |
| 可读性      | 	极佳（1, 2, 3...）		         | 	    极差（550e8400-e29b...）            |
| 写入性能     | 	极高（顺序追加，索引紧凑）		          | 较低（随机写入导致索引页频繁分裂）                    |
| 安全性      | 	低（容易通过 ID 猜出业务量、爬取数据）    | 高（不可预测，难以被枚举）                        |
| 分布式环境  	 | 难（需要中心化分配器，易冲突）           | 极易（本地生成，几乎不可能冲突）                     |
| 排序属性     | 	天然有序			                  | 		    默认无序（UUID v1/v7 部分有序，v4 随机）    |
