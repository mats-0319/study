# Go Code Style

记录我的go代码风格

## 判断error，是否将函数调用写在if行内

``` go 
// style 1
err := f()
if err != nil {}

// style 2
if err := f(); err != nil {}
```

- 风格2可以限定`err`的作用域，让代码更清晰
- 风格1在多返回值等场景下更简洁，举个例子：
  ``` go
  count, users, err := db.listUser()
  if err != nil {}
  ```
  在上例中，风格2需要提前定义全部左值，并且`err`的作用域也没有限制住

综上所述，多返回值场景下，使用风格1；简单单一返回值场景下，使用风格2

- 简单：指返回值为error、ok等不包含业务数据的类型，且返回值最多判断一下、后续不会再使用的场景

## 命名的大小写问题

如果一个名字或者名字中的一段，它已经是缩写了，则它整体（全部字母）的大小写应统一。

- 因为go语言标签首字母的大小写具有额外含义（是否可导出），所以只要求整体大小写统一
- 举几个例子：`ID`/`MFA`/`HTTP`/`TOTP`，它们本身已经是缩写了，所以程序中不应看到`Mfa`/`Totp`此类写法
