# markdown语法与关键字

## 语法

- 标题：1～6个`#`表示H1～H6
- 加粗：使用`**`包围**想要加粗的内容**
- 斜体：使用`*`包围*想要使用斜体的内容*
    - 加粗和斜体还可以使用`_`
    - `***`包围的内容将被识别为***加粗+斜体***
- 删除线：使用`~~`包围~~想要应用删除线的内容~~
- 引用：在行首使用`>`（符号和内容中间可以没有空格，但建议有）

> 想要引用的内容

- 列表：在行首（允许缩进）使用指定符号+空格
    - 无序列表：`-`、`+`、`*`
    - 有序列表：数字+`.`，例如`1.`
    - 任务列表：形如`- [ ]`/`- [x]`，表示列表中的一项任务未完成/已完成
- 代码：
    - 行内代码：使用` `` ` 包围`想要表示为代码的内容`
    - 多行代码：使用` ``` `包围多行代码并可指定语言

```ts
// some ts code
let i = 10
```

- 链接：形如`[github](https://github.com)`
- 图片：形如`![替代文字](图片地址)`
- 分割线：使用`---`或`***`

### github支持语法

- 下标：`<sub> </sub>`，例如：a<sub>1</sub>+a<sub>2</sub>=1
- 上标：`<sup> </sup>`，例如：a<sup>2</sup>+b<sup>2</sup>=1
- 下划线：`<ins> </ins>`，例如：some <ins>underlined</ins> text
- 颜色：`#ffffff`，例如：`rgb(210,209,186)`/`rgb(240,239,226)`
- 相对链接：可以通过链接的形式，引入基于当前markdown文件路径的其他文件。例如[link syntax](markdown.md)

## 关键字

从语法中提取常用markdown关键字：

- `#`、`*`、`-`、`+`、`[]`、`()`、`~`、`<>`

关键字会在渲染过程中被赋予特殊含义，所以在常规使用中应避免使用它们。
- 举个例子，在一条命令中，我们通常使用`[]`表示一个参数是可选的（例如：`git push [repository name] [branch name]`），
而在markdown中，则应使用其他表示方式或将整条命令标记为代码

## 技巧

- 转义：使用`\`可以显示字符原本的样子，例如`\*`将打印一个星号而不是斜体
- 换行：纯文字的换行需要在一行末尾添加复数空格或`<br/>`（html换行符）
